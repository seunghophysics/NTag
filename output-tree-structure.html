mue

This tree contains the information of signal candidates retrieved from the MUE bank. This tree will be filled only if the input file (ZEBRA file) has a MUE bank, which can be created if the ATMPD decay-e search routine muechk (or APFit) is applied before feeding it to NTag. MUE candidates with time (FitT) overlapping with the neutron search time interval specified by -TMIN and -TMAX commands (see here for details) will be pruned automatically to prevent double-counting a single signal by MUE and NTag.

Branch name Description
NCandidates Total number of MUE candidates

Each entry in the following branches is an array with a size of the number of MUE candidates in each event. Vertex fit is skipped for MUE candidates that are too close in time.

Branch name Description
DWall Distance to nearest wall (cm)
